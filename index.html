Bash Commands

This page explains how to use Bash Commands and defines what they do.

  1. Git Add
    • Adds a file to the staging area.
    • Example to add single file: git add index.html
    • Example to add all unstaged files to the staging area: git add .
  2. Git Commit
    • Records a snapshot of the file permanently in the version history.
    • Example: git commit -m 'comment explaining changes since last commit'
    • Another method: git commit
    • Click "i" for insert
    • Enter comment explaining changes since last commit
    • Click escape
    • Click ":wq" to return to git bash window
  3. Git Clone
    • Copies a repository from an existing URL.
    • Example: git clone https://github.com/example/example.git
  4. Git Config
    • Sets the author name and email address to be used with the commit command.
    • Example: git config --global user.name 'username'
    • Example: git config --global user.email 'user@email.com'
  5. Git Fetch
    • Checks for available changes from the remote repository, but does not actually transfer files.
    • This is useful to know what changes you will see before performing a pull.
    • Example: git fetch
  6. Git Merge
    • Merges the specified branch's history into the current branch.
    • Example: git merge branchName
  7. Git Move
    • Sets the author name and email address to be used with the commit command.
    • Example: git Config --global user.name 'username'
    • Example: git Config --global user.email 'user@email.com'
  8. Git Pull
    • Checks for and brings available changes from the remote repository into your local copy.
    • Example: git pull https://github.com/example/example.git
  9. Git Push
    • Sends the committed changes of the master branch (or other specified branch) to the associated remote repository.
    • Example: git push variableName branchName
  10. Git Remote
    • Connects your local repository to the remote server.
    • Example: git remote add variableName https://github.com/example/example.git
  11. Git Remove
    • Removes individual files or a collection of files from the staging area and the working directory of the current branch.
    • Example: git rm exampleFile.txt
  12. Git Status
    • Lists all files, stating files that have changes to be committed, or files that need to be added to staging area.
    • Example: git status
